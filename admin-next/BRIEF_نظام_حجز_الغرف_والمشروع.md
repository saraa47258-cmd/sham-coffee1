## ٢. تحليل نظام حجز الغرف

### ٢.١ الغرض من النظام
- إدارة **غرف خاصة** للمقهى (جلسات، عائلات، VIP).
- تتبع حالة كل غرفة: **متاحة**، **محجوزة**، **مشغولة**.
- ربط الغرف بـ **طلبات** (مشروبات/أطباق) و **تسعير** حسب نوع الغرفة وجنس الحاجز.

### ٢.٢ نموذج البيانات – الغرفة (Room)

| الحقل | النوع | الوصف |
|-------|-------|--------|
| `id` | string | معرّف فريد |
| `roomNumber` | string | رقم الغرفة (مثال: 1، VIP1، غ1) |
| `name` | string? | اسم اختياري (غرفة العائلات، VIP، إلخ) |
| `status` | enum | `available` \| `reserved` \| `occupied` |
| `notes` | string? | ملاحظات |
| `isActive` | boolean | نشطة أم معطّلة |
| `activeOrderId` | string? | معرّف الطلب النشط عند `occupied` |
| `reservedBy` | string? | اسم الحاجز عند `reserved` |
| `reservedAt` | string? | وقت الحجز (ISO) |
| `priceType` | enum | `free` \| `fixed` \| `gender` |
| `hourlyRate` | number? | سعر الساعة (عند `fixed`) |
| `malePrice` | number? | سعر الذكور (عند `gender`) – افتراضي 3 ر.ع |
| `femalePrice` | number? | سعر الإناث (عند `gender`) – 0 = مجاني |

### ٢.٣ حالات الغرفة (Status)

| الحالة | العربية | المعنى |
|--------|---------|--------|
| `available` | متاحة | يمكن حجزها أو فتح طلب مباشرة |
| `reserved` | محجوزة | محجوزة باسم معيّن، لم يُفتح طلب بعد |
| `occupied` | مشغولة | يوجد طلب نشط مرتبط بالغرفة |

### ٢.٤ أنواع التسعير (priceType)

| النوع | الوصف | الحقول المستخدمة |
|-------|--------|-------------------|
| `free` | مجاني | لا يُستخدم سعر |
| `fixed` | سعر ثابت | `hourlyRate` (ر.ع/ساعة) |
| `gender` | حسب الجنس | `malePrice`، `femalePrice` (مثلاً ذكور 3 ر.ع، إناث 0) |

## ٣. مسار العمل (Workflow) – من الحجز إلى الإغلاق

### ٣.١ عند كون الغرفة **متاحة**
1. **حجز (Reserve):**
   - المستخدم يفتح تفاصيل الغرفة → "حجز".
   - يدخل **اسم الحاجز** → تأكيد.
   - يتم استدعاء `setRoomStatus(roomId, 'reserved', null, reservedBy)`.
   - تُحدَّث `reservedBy` و `reservedAt`.

2. **طلب جديد (بدون حجز مسبق):**
   - من تفاصيل الغرفة → "طلب جديد" → ينتقل إلى `/admin/staff-menu?roomId=...`.
   - من الكاشير يتم اختيار الغرفة ونوع الطلب `room`، وإضافة المشروبات/الأصناف.
   - عند الدفع/إتمام الطلب:
     - يُنشأ الطلب في Firebase.
     - يتم `setRoomStatus(roomId, 'occupied', orderId)`.
     - الطلب يُربط بالغرفة عبر `roomId` و `activeOrderId`.

### ٣.٢ عند كون الغرفة **محجوزة**
- تظهر "محجوزة لـ {reservedBy}" مع وقت الحجز.
- يمكن تحريرها → "تحرير الغرفة" → `status = available`.
- أو عند فتح طلب فعلي (مثلاً من الكاشير) وربطه بالغرفة، تنتقل إلى `occupied` كما أعلاه.

### ٣.٣ عند كون الغرفة **مشغولة**
- يوجد طلب نشط (`activeOrderId`).
- الواجهة تعرض:
  - رقم الطلب، الوقت، عناصر الطلب، الإجمالي.
  - إن وُجد: جنس الحاجز (ولد/بنت) وسعر الغرفة (`roomGender`, `roomPrice`).
- إجراءات متاحة:
  - **فتح في الكاشير:** `/admin/cashier?roomId=...&orderId=...` لإكمال الدفع أو إضافة أصناف.
  - **إغلاق الطلب وتحرير الغرفة:**
    - استدعاء `closeRoomOrder(orderId, roomId)`:
      - تحديث الطلب: `status = completed`, `paymentStatus = paid`.
      - `setRoomStatus(roomId, 'available')` وإزالة `activeOrderId` و `reservedBy` و `reservedAt`.

### ٣.٤ إنشاء طلب غرفة من الكاشير
- في `PaymentPanel` (الكاشير):
  - نوع الطلب: `room`.
  - اختيار الغرفة من القائمة.
  - إذا كانت الغرفة `gender`:
    - اختيار **ولد** أو **بنت**.
    - يُحسب `roomPrice` من `malePrice` أو `femalePrice` ويُضاف إلى إجمالي الطلب.
- عند إتمام الطلب يُستخدم `createPOSOrder` من `lib/pos.ts`:
  - يُنشأ الطلب مع `orderType: 'room'`, `roomId`, `roomNumber`, `roomGender`, `roomPrice`.
  - يتم ربط الغرفة وتحديث حالتها إلى `occupied` كما سبق.

## ٤. الصفحات والمكونات المرتبطة بالغرف

### ٤.١ `/admin/rooms` – إدارة الغرف
- **الوظيفة:** عرض كل الغرف، إحصائيات (إجمالي، متاحة، محجوزة، مشغولة)، فلاتر (حالة، نشطة/غير نشطة، بحث).
- **الإجراءات:**
  - إضافة غرفة (`RoomModal`).
  - تعديل غرفة (نفس المودال).
  - تفعيل/إلغاء تفعيل غرفة.
  - حذف غرفة (غير مسموح إذا كان لها طلب نشط).
- **عرض البيانات:**
  - جدول (ديسكتوب) أو بطاقات (موبايل/تابلت).
  - رقم الغرفة، الاسم، التسعير، الحالة، الطلب الحالي، المدة، جنس الحاجز إن وُجد.

### ٤.٢ `RoomDetailsModal`
- يفتح عند النقر على غرفة.
- يعرض: التسعير، الملاحظات، "محجوزة لـ"، الطلب النشط (عناصر، إجمالي، جنس، سعر الغرفة).
- أزرار: حجز، طلب جديد، تحرير الغرفة، إغلاق الطلب وتحرير الغرفة، فتح في الكاشير، طباعة.

### ٤.٣ `/admin/room-orders` – طلبات الغرف
- **الوظيفة:** قائمة طلبات الغرف فقط.
- **الفلاتر:** نطاق زمني (اليوم، أسبوع، شهر، سنة، مخصص)، غرفة، حالة الطلب، حالة الدفع، بحث.
- **الإحصائيات:** إجمالي الطلبات، إجمالي المبيعات، مدفوع / غير مدفوع، معلق.
- **الإجراءات:** عرض تفاصيل الطلب (`OrderDetailsDrawer`)، تحديث الحالة، دفع، إلغاء. عند الدفع أو الإلغاء يتم استدعاء `closeRoomOrder` إن لزم.

### ٤.٤ الكاشير والمنيو
- **staff-menu:** أخذ الطلب من غرفة معيّنة (عبر `roomId` في الرابط).
- **cashier:** اختيار غرفة، تحديد نوع الطلب `room`، جنس الحاجز عند التسعير `gender`، وإتمام الطلب أو الدفع.

## ٥. دوال قاعدة البيانات المتعلقة بالغرف

| الدالة | الوصف |
|--------|--------|
| `listenToRooms(cb)` | استماع حي للتحديثات على الغرف |
| `getRoom(roomId)` | جلب غرفة بالمعرّف |
| `getRooms()` | جلب كل الغرف |
| `createRoom(room)` | إنشاء غرفة جديدة |
| `updateRoom(roomId, updates)` | تحديث غرفة |
| `deleteRoom(roomId)` | حذف غرفة |
| `setRoomStatus(roomId, status, activeOrderId?, reservedBy?)` | تغيير الحالة (متاحة/محجوزة/مشغولة) |
| `getRoomWithOrder(roomId)` | غرفة مع الطلب النشط إن وُجد |
| `getOrdersByRoom(roomId)` | كل الطلبات المرتبطة بغرفة |
| `getRoomOrders()` | كل طلبات الغرف |
| `getRoomOrdersByDateRange(range)` | طلبات الغرف ضمن نطاق زمني |
| `createRoomOrder(roomId, orderData)` | إنشاء طلب غرفة وربطه بالغرفة (`occupied`) |
| `closeRoomOrder(orderId, roomId)` | إغلاق الطلب وتحويل الغرفة إلى `available` |

## ٦. قواعد العمل المهمة

1. **حذف غرفة:** لا يُسمح إذا كان لها `activeOrderId`. يجب إغلاق الطلب أولاً.
2. **إلغاء تفعيل غرفة:** لا يُسمح إذا كان لها طلب نشط.
3. **تحرير غرفة مشغولة:** يتم عبر "إغلاق الطلب وتحرير الغرفة" أو عبر دفع/إلغاء الطلب من صفحة طلبات الغرف.
4. **تسعير حسب الجنس:** يُطبَّق فقط عندما `priceType === 'gender'`؛ يلزم اختيار ولد/بنت في الكاشير قبل إتمام الطلب.

## ٧. ملخص سريع – نظام حجز الغرف

| العنصر | الوصف |
|--------|--------|
| **الغرف** | إدارة غرف خاصة بتسعير (مجاني / ثابت / حسب الجنس) |
| **الحالات** | متاحة ← حجز أو طلب مباشر ← محجوزة/مشغولة ← إغلاق طلب ← متاحة |
| **الحجز** | اسم الحاجز، تحديث `reservedBy` و `reservedAt` |
| **الطلبات** | طلبات `orderType: 'room'` مرتبطة بـ `roomId`، مع `roomGender` و `roomPrice` عند الحاجة |
| **الصفحات** | `/admin/rooms`، `/admin/room-orders`، الكاشير، المنيو |
| **البيانات** | Firebase Realtime DB – مجموعات `rooms` و `orders` |
